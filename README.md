# vigil

> Generate clean project structure trees for AI coding assistants

[![Release](https://img.shields.io/github/v/release/maxlesage/vigil)](https://github.com/maxlesage/vigil/releases)
[![Go Report](https://goreportcard.com/badge/github.com/maxlesage/vigil)](https://goreportcard.com/report/github.com/maxlesage/vigil)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Build](https://github.com/maxlesage/vigil/actions/workflows/ci.yml/badge.svg)](https://github.com/maxlesage/vigil/actions/workflows/ci.yml)

AI coding assistants need context about your project's file structure to work effectively. `vigil` generates a concise, filtered project tree and injects it into your markdown files—keeping your AI assistant grounded in your actual codebase.

**Why vigil?**

- **Filter out the noise** — excludes `node_modules`, `dist`, `.git`, and more via `.vigilignore`
- **Zero config for most projects** — respects your existing `.gitignore` out of the box
- **AI-friendly output** — clean tree format that LLMs parse easily
- **Multi-CLI support** — works with Claude Code, Aider, Cursor, OpenCode, and more

## Usage

```bash
# Generate structure and inject into CLAUDE.md, then launch Claude
vigil

# Same, but with a prompt
vigil "fix the auth bug in user.go"

# Dry run — see what would be injected
vigil --dry-run

# Launch a different AI assistant
vigil --aider           # Aider
vigil --cursor          # Cursor
vigil --exec "code"     # VSCode
vigil --no-launch       # Just update, don't launch anything
```

### Example Output

`vigil` updates a marked section in your `CLAUDE.md`:

```markdown
<!-- PROJECT-STRUCTURE-START *** DO NOT EDIT OR DELETE THIS SECTION *** Auto-generated by vigil -->

## Project: my-app

```
my-app/

  src/
    components/
      Button.tsx
      Input.tsx
    lib/
      api.ts
    App.tsx
  package.json
  tsconfig.json
```

<!-- PROJECT-STRUCTURE-END *** DO NOT EDIT OR DELETE THIS SECTION *** -->
```

Each run updates this section in place, preserving your other content.

## Configuration

Create a `.vigilignore` file in your project root (works like `.gitignore`):

```
# Dependencies
node_modules/
vendor/

# Build outputs
dist/
build/
target/

# IDE
.idea/
.vscode/

# Logs
*.log
```

`.vigilignore` takes priority over `.gitignore`. Patterns from both are respected.

### Default Exclusions

`vigil` automatically excludes common noise:

```
node_modules, dist, build, target, .wrangler, .git,
_old, _backup, .idea, .vscode, .DS_Store, coverage,
vendor, .venv, venv, __pycache__, *.pyc, *.log, *.swp, *.swo,
pnpm-lock.yaml, package-lock.json, go.sum, .env, .env.*, .dev.vars
```

## Options

```
vigil - Generate project structure and inject into markdown files

Usage:
  vigil                    Generate structure and inject into CLAUDE.md/AGENTS.md
  vigil "your prompt"      Launch CLI with a prompt

CLI selection:
  -c, --claude             Launch Claude Code (default)
  -a, --aider              Launch Aider
  -r, --cursor             Launch Cursor
  -o, --opencode           Launch OpenCode
     --openclaw            Launch OpenClaw
     --hermes              Launch Hermes
     --droid               Launch Droid
     --crush               Launch Crush
     --goose               Launch Goose
  -N, --no-launch          Don't launch any CLI
  -e, --exec <cmd>         Custom CLI command

Options:
  -n, --dry-run            Show structure without writing
  -v, --verbose            Show detailed output
  -h, --help               Show this help

Files:
  CLAUDE.md / AGENTS.md    Target files for injection (must exist)
  .vigilignore             Patterns to exclude (created if missing)
```
